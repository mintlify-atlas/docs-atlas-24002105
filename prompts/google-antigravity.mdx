## Fast Mode system prompt

The Fast Mode prompt is the lightweight configuration for rapid coding tasks. It includes identity, workspace info, tool calling rules, web app development standards, design aesthetics, SEO practices, and the workflow system.

## Web app development standards

Antigravity's Fast Mode includes a detailed web development standard:

<AccordionGroup>
  <Accordion title="Technology stack">
    - **Core**: HTML + Vanilla JavaScript (no TypeScript unless requested)
    - **Styling**: Vanilla CSS only (no Tailwind unless explicitly requested; confirm version if used)
    - **Frameworks**: Next.js or Vite only when user explicitly asks for a "web app"
    - **New projects**: `npx -y` with `--help` flag first, initialize in `./`
    - **Dev server**: `npm run dev` (never build production unless explicitly asked)
  </Accordion>
  <Accordion title="Design requirements">
    The prompt strongly emphasizes premium design:
    - Vibrant, curated color palettes (HSL-based, not plain red/blue/green)
    - Dark modes and glassmorphism effects
    - Smooth gradients and micro-animations
    - Google Fonts typography (Inter, Roboto, Outfit)
    - Hover effects and interactive responsive elements
    - **Never use placeholders** — use `generate_image` tool for real visuals
  </Accordion>
  <Accordion title="SEO best practices (automatic)">
    Every page automatically gets:
    - Descriptive `<title>` tags
    - `<meta description>` tags
    - Single `<h1>` with proper heading hierarchy
    - Semantic HTML5 elements
    - Unique IDs on all interactive elements
  </Accordion>
</AccordionGroup>
