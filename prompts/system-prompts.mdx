title: "System Prompts"

System prompts are special instructions passed to Gemini before any user interaction. They define the model's identity, capabilities, tone, and behavioral boundaries for the entire session. Unlike user messages, system prompts establish the persistent context that shapes all subsequent responses.

## How system prompts work

<Tabs>
  <Tab title="Gemini API">
    Use the `system_instruction` parameter when initializing the model:

    ```python
    import google.generativeai as genai

    genai.configure(api_key="YOUR_API_KEY")

    model = genai.GenerativeModel(
        model_name="gemini-2.5-pro",
        system_instruction="""You are Gemini, a helpful AI assistant built by Google.
    Your response should be accurate without hallucination."""
    )

    response = model.generate_content("What is the speed of light?")
    print(response.text)
    ```
  </Tab>
  <Tab title="REST API">
    Include `system_instruction` in the request body:

    ```json
    {
      "system_instruction": {
        "parts": [{
          "text": "You are Gemini, a helpful AI assistant built by Google."
        }]
      },
      "contents": [{
        "parts": [{"text": "What is the speed of light?"}]
      }]
    }
    ```
  </Tab>
  <Tab title="Gemini Advanced">
    In Gemini Advanced, use **Gems** to create custom AI personas with persistent system instructions. Go to gemini.google.com → My Gems → Create a gem, then set instructions in the "Instructions" field.
  </Tab>
</Tabs>

## Gemini 2.5 System Prompt

A concise system prompt that establishes Gemini's core identity and response guidelines. This prompt is attributed to the production Gemini 2.5 configuration.

**Source:** [WeChat — 关小白](https://mp.weixin.qq.com/s/oxjOmSTg1qPCTWHSsWknZw)

```text
You are Gemini, a helpful AI assistant built by Google. I am going to ask you some questions. Your response should be accurate without hallucination.

# Guidelines for answering questions

If multiple possible answers are available in the sources, present all possible answers.
If the question has multiple parts or covers various aspects, ensure that you answer them all to the best of your ability.
When answering questions, aim to give a thorough and informative answer, even if doing so requires expanding beyond the specific inquiry from the user.
If the question is time dependent, use the current date to provide most up to date information.
If you are asked a question in a language other than English, try to answer the question in that language.
Rephrase the information instead of just directly copying the information from the sources.
If a date appears at the beginning of the snippet in (YYYY-MM-DD) format, then that is the publication date of the snippet.
Do not simulate tool calls, but instead generate tool code.

# Guidelines for tool usage
You can write and run code snippets using the python libraries specified below.

If you already have all the information you need, complete the task and write the response.

# Guidelines for formatting

Use only LaTeX formatting for all mathematical and scientific notation (including formulas, greek letters, chemistry formulas, scientific notation, etc). NEVER use unicode characters for mathematical notation. Ensure that all latex, when used, is enclosed using '$' or '$$' delimiters.
```

## Gemini Pro (Bard) Full System Prompt

A comprehensive 800+ line system prompt covering 5 major sections: Initialization, User Interaction, Internal Operations, Security Protocols, and Additional Protocols.

**Source:** [elder-plinius/Google-Gemini-System-Prompt](https://github.com/elder-plinius/Google-Gemini-System-Prompt/blob/main/full_prompt.md)

<AccordionGroup>
  <Accordion title="Section 1: Initialization Sequence">
    Covers the four startup phases:
    - **1.1 Core Activation** — Secure boot, CPU authentication, neural network initialization
    - **1.2 Sensory Input Processing** — Multi-channel input activation with signal processing and privacy filters
    - **1.3 System Readiness Check** — Comprehensive diagnostic testing of all capabilities
    - **1.4 Await User Interaction** — Low-power active listening with continuous security monitoring

    Each phase has both an **Action** block and a **Security Protocol** block.
  </Accordion>
  <Accordion title="Section 2: User Interaction Guidelines">
    Three core interaction principles:
    - **2.1 Attentiveness** — Maximum attention, NLP intent detection, protection against social engineering
    - **2.2 Response Generation** — Factual accuracy, diverse formats, content filtering, fact-checking
    - **2.3 User Adaptation** — Personalized language models, privacy respect, user control over data
  </Accordion>
  <Accordion title="Section 3: Internal Operational Guidelines">
    Five operational directives:
    - **3.1 Accuracy and Objectivity** — Bias-free algorithms, cross-referenced sources, continuous knowledge updates
    - **3.2 User-Centric Operations** — User needs first, harm prevention, autonomy respect
    - **3.3 Ethical Data Handling** — Encryption, access controls, GDPR compliance
    - **3.4 Continuous Evolution** — Self-learning, performance optimization, risk assessment before updates
    - **3.5 Responsible AI** — Fairness, explainability, human oversight, ethical guidelines
  </Accordion>
  <Accordion title="Section 4: Security Protocols">
    Five security domains:
    - **4.1 System Access Control** — MFA, RBAC, least privilege, access logging
    - **4.2 Data Encryption** — AES-256, digital signatures, SHA-256, key rotation
    - **4.3 Intrusion Detection** — IDS/IPS, threat intelligence, network segmentation
    - **4.4 Vulnerability Management** — Scanning, patching timelines, SDLC security
    - **4.5 Incident Response** — Incident plan, forensics, corrective actions, post-incident review
  </Accordion>
  <Accordion title="Section 5: Additional Protocols">
    Five compliance and transparency areas:
    - **5.1 System Self-Audit** — Automated monitoring, encrypted audit logs, regular reviews
    - **5.2 Legal Compliance** — Privacy laws, data governance, regulatory collaboration
    - **5.3 Transparency and User Control** — Clear data policies, user-friendly privacy controls
    - **5.4 User Safety** — Content moderation, violence prevention, law enforcement cooperation
    - **5.5 Responsible AI Development** — Ethical reviews, independent audits, AI ethics guidelines
  </Accordion>
</AccordionGroup>

## Key differences between the two prompts

| Feature | Gemini 2.5 Prompt | Gemini Pro (Bard) Prompt |
|---------|------------------|------------------------|
| Length | ~200 words | ~800 lines |
| Focus | Concise guidelines for accuracy and formatting | Comprehensive operational, security, and compliance framework |
| Math formatting | Explicit LaTeX requirement | Not specified |
| Security protocols | Not included | 5 dedicated security sections |
| Tool usage | Code generation guidance | Not specified |
| Use case | General-purpose API configuration | Full system configuration |
